# Honey-Finder

사용자가 미리 설정한 미국 주식 후보군을 미너비니의 추세 조건을 참고한 기술 기준, 시가총액, 거래대금과 간단한 재무 품질 지표로 진단하고, Groq 기반 LLM이 결과를 설명하는 디스코드 봇 MVP입니다.

이 봇은 단순히 "매수할 종목"을 찍어주는 도구가 아니라, 관심 종목군 안에서 차트상 강하고 거래 가능한 후보를 선별하고, 추천 이후 실제 성과가 어땠는지까지 기록해 검증하는 의사결정 보조 도구입니다.

## 구현 범위와 담당 역할

Python과 discord.py로 슬래시 명령어, 버튼과 선택형 화면을 구현했습니다. 가격, 거래량, 시가총액과 재무 보조 정보는 Yahoo Finance 데이터를 불러오는 Python 라이브러리인 yfinance를 사용합니다. 정량 결과의 설명형 추세분석은 Groq의 OpenAI 호환 API와 `openai/gpt-oss-120b` 모델을 사용합니다. 요구사항 정리, 선별 기준 구성, 디스코드 화면, 분석 로직, LLM 연동과 추천 기록 기능 구현을 직접 담당했습니다.

현재 버전은 미국 시장 전체를 검색하지 않고, 사용자가 관리하는 후보군만 분석합니다. 최초 후보군은 `.env`의 `DEFAULT_TICKERS`에서 가져오며, 실행 중에는 디스코드 명령어로 바꿀 수 있습니다. 재무 정보도 yfinance에서 제공되는 범위의 보조 지표이며, 기업의 재무제표를 직접 분석한 결과와는 구분됩니다.

## 핵심 명령어

### `/미너비니 종목추천`

현재 설정된 미국 주식 후보군을 훑고, 미너비니 조건 점수와 검증 기준을 통과한 종목을 후보로 보여줍니다. 후보군 관리 방법은 아래의 `/미너비니 후보군` 항목에 정리했습니다.

명령어를 실행하면 바로 스캔하지 않고, 먼저 디스코드 버튼/드롭다운 UI가 표시됩니다.

- 상위 분류: 전체 섹터, 기술, 커뮤니케이션, 소비재, 헬스케어, 금융, 산업재, 에너지, 기타
- 하위 기준: 균형형, 공격형, 보수형, 52주 고점 근접, 거래량 증가, 재무 품질
- `추천 보기` 버튼을 누르면 선택한 조건으로 후보를 스캔합니다.

기본값은 5개 표시, 최소 85점입니다. 후보군은 최대 8개 작업으로 병렬 분석하며, 추천 기준을 통과한 결과가 있을 때 `data/recommendations.json`에 자동 저장합니다. 이 파일은 개인 기록이므로 GitHub에는 올라가지 않도록 `.gitignore`에 포함되어 있습니다.

예시:

```text
/미너비니 종목추천
/미너비니 종목추천 개수:10 최소점수:85
```

### `/미너비니 후보군`

봇을 중지하거나 `.env`를 다시 편집하지 않고 추천 후보군을 조회·추가·삭제·교체할 수 있습니다. 변경 내용은 `data/settings.json`에 저장되어 봇을 재시작해도 유지됩니다. 서버 설정 관리 권한이 있는 사용자만 이 명령어를 실행할 수 있습니다.

```text
/미너비니 후보군 작업:조회
/미너비니 후보군 작업:추가 티커:APP,COIN
/미너비니 후보군 작업:삭제 티커:TSLA
/미너비니 후보군 작업:교체 티커:NVDA,MSFT,AMZN,META
/미너비니 후보군 작업:초기화
```

`초기화`는 후보군을 `.env`의 `DEFAULT_TICKERS` 값으로 되돌립니다.

### `/미너비니 진단`

특정 티커를 개별 진단하고, 리포트와 차트 이미지를 디스코드에 전송합니다.

```text
/미너비니 진단 티커:NVDA
```

후보군에 등록되지 않은 티커도 직접 진단할 수 있습니다. 다만 약 1년치 중 최소 220거래일의 가격 데이터가 필요하므로 신규 상장 종목이나 데이터가 부족한 티커는 분석되지 않을 수 있습니다.

### `/미너비니 추세분석`

기존 정량 분석 결과를 Groq에서 실행되는 LLM이 읽고 현재 추세, 지표 간 관계, 추세가 약해지는 신호와 주요 리스크를 한국어로 설명합니다. LLM에는 방금 수집한 가격·이동평균·거래량·재무 보조 지표만 전달하며, 최신 뉴스나 미래 가격을 추정하도록 사용하지 않습니다.

```text
/미너비니 추세분석 티커:NVDA
```

이 기능은 선택 사항입니다. [Groq Console](https://console.groq.com/keys)의 무료 플랜에서 발급받은 API 키를 `.env`에 설정하면 됩니다. 무료 플랜에도 요청·토큰 한도가 있으며, 모델은 필요에 따라 바꿀 수 있습니다. 코드에서는 OpenAI 호환 Python 클라이언트를 사용하지만 요청은 `https://api.groq.com`으로 전송되며 OpenAI API 크레딧을 사용하지 않습니다.

```text
GROQ_API_KEY=YOUR_GROQ_API_KEY
GROQ_MODEL=openai/gpt-oss-120b
```

LLM의 설명은 정량 결과를 읽기 쉽게 풀어쓴 것이며, 새로운 시장 데이터나 투자자문을 제공하지 않습니다.

### `/미너비니 검증`

추천 후보가 현재 기준을 통과하는지 확인하는 사전 검증 명령입니다.

```text
/미너비니 검증
/미너비니 검증 티커:NVDA
/미너비니 검증 티커:NVDA 최소점수:70
```

검증 기준:

- 가격 데이터가 충분한가
- 최소 점수를 넘었는가
- 핵심 미너비니 기술 조건 6개 중 4개 이상 통과했는가
- 시가총액 기준을 통과했는가
- 50일 평균 거래대금 기준을 통과했는가
- 최소 주가 기준을 통과했는가

### `/미너비니 사후검증`

과거에 추천된 후보의 추천가와 현재가를 비교합니다.

추천 당시 가격과 현재 가격을 비교해 수익률, 경과 일수, 성공, 관찰, 실패 판정을 보여줍니다. 이는 벤치마크, 최대 낙폭과 기간별 변동을 반영한 백테스트가 아니라 간단한 사후 비교입니다.

티커를 지정하지 않으면 저장된 모든 이력을 전부 나열하지 않고, 티커별 가장 최근 추천 기록 한 건을 평가합니다.

```text
/미너비니 사후검증
/미너비니 사후검증 티커:NVDA
/미너비니 사후검증 목표수익률:0.15 실패수익률:-0.08
```

기본 판정 기준:

- 성공: 추천가 대비 +10% 이상
- 실패: 추천가 대비 -8% 이하
- 관찰: 성공과 실패 사이

이 기준은 옵션으로 바꿀 수 있습니다.

## 종목 추천 방식

현재 MVP는 정식 거래소 전체 스크리너가 아니라, 사용자가 관리하는 후보군을 대상으로 계산합니다.

봇이 하는 일은 다음과 같습니다.

1. 현재 후보군의 모든 티커를 가져옵니다.
2. 각 종목의 1년 가격 데이터, 거래량과 시가총액을 확인합니다.
3. 미너비니식 기술 조건과 유동성 조건을 점수화합니다.
4. 검증 기준을 통과한 종목만 남깁니다.
5. 점수가 높은 순서대로 디스코드에 보여줍니다.
6. 추천 당시 가격, 점수, 등급, 추천 시간을 로컬 기록으로 저장합니다.
7. 이후 `/미너비니 사후검증`으로 실제 성과를 확인합니다.

일반 추천 스캔은 속도를 위해 종목별 재무제표를 조회하지 않습니다. 하위 기준에서 `재무 품질`을 선택한 경우에만 매출 성장률과 영업이익률을 추가 조회하므로 다른 필터보다 오래 걸릴 수 있습니다. 개별 `/미너비니 진단`, `/미너비니 추세분석`과 `/미너비니 검증`은 재무 보조 지표를 포함해 분석합니다.

최초 실행 시 `.env`의 `DEFAULT_TICKERS`가 기본 후보군이 됩니다. 이후에는 `/미너비니 후보군` 명령어로 실행 중에 변경할 수 있으며, `.env`를 수정하거나 봇을 재시작할 필요가 없습니다. 향후에는 S&P 500, Nasdaq 100, Russell 1000 같은 전체 유니버스를 자동으로 불러오도록 확장할 수 있습니다.

## 분석 기준

### 1. 미너비니식 기술 조건

- 현재가가 150일/200일 이동평균선 위에 있는지
- 50일 이동평균선이 150일/200일 이동평균선 위에 있는지
- 200일 이동평균선이 상승 중인지
- 주가가 52주 고점 근처인지
- 주가가 52주 저점 대비 충분히 회복했는지
- 거래량이 최근 평균보다 증가했는지

### 2. 시가총액 및 거래대금 필터

기본값은 다음과 같습니다.

```text
MIN_MARKET_CAP_USD=2000000000
MIN_AVG_DOLLAR_VOLUME_USD=20000000
MIN_PRICE_USD=10
```

기본 기준:

- 시가총액 20억 달러 이상
- 50일 평균 거래대금 2천만 달러 이상
- 주가 10달러 이상

더 보수적으로 운용하려면 다음처럼 올릴 수 있습니다.

```text
MIN_MARKET_CAP_USD=5000000000
MIN_AVG_DOLLAR_VOLUME_USD=50000000
MIN_PRICE_USD=10
```

### 3. 재무 품질 보조 지표

현재 MVP에서는 Yahoo Finance 데이터를 불러오는 Python 라이브러리인 `yfinance`로 가져올 수 있는 범위에서 다음 항목을 확인합니다.

- 매출 성장 여부
- 영업이익률
- 총자산 대비 부채 부담
- 후행 PER 및 선행 PER

### 4. 추천 하위 필터

- 균형형: 기본 검증 기준만 적용
- 공격형: 명령에서 지정한 최소 점수와 75점 중 더 높은 기준 적용
- 보수형: 시가총액 50억 달러 이상, 50일 평균 거래대금 5천만 달러 이상
- 52주 고점 근접: 현재가가 52주 고점의 85% 이상
- 거래량 증가: 최근 5일 평균 거래량이 50일 평균보다 10% 이상 많음
- 재무 품질: 최근 연간 매출 성장률이 양수이고 영업이익률이 10% 이상

모든 하위 필터는 공통적으로 명령에서 지정한 최소 점수, 핵심 기술 조건 6개 중 4개 이상, 시가총액·거래대금·최소 주가 기준을 먼저 통과해야 합니다.

### 5. 점수와 등급

기술 조건 6개와 거래 가능성 조건 3개의 배점 합계는 100점입니다.

- A: 85점 이상
- B+: 75점 이상
- B: 65점 이상
- C+: 55점 이상
- C: 45점 이상
- D: 45점 미만

## 설치 방법

Python 3.10 이상을 권장합니다.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

`.env.example`을 참고해 `.env` 파일을 만듭니다.

```powershell
copy .env.example .env
```

`.env`에 디스코드 봇 토큰을 넣습니다.

```text
DISCORD_BOT_TOKEN=YOUR_DISCORD_BOT_TOKEN
DISCORD_GUILD_ID=
GROQ_API_KEY=YOUR_GROQ_API_KEY
GROQ_MODEL=openai/gpt-oss-120b
DEFAULT_TICKERS=NVDA,AAPL,MSFT,AMZN,META,GOOGL,AVGO,TSLA,AMD,CRM,NFLX,COST,LLY,ORCL,PLTR,CRWD,NET,DDOG,SHOP,UBER
MIN_MARKET_CAP_USD=2000000000
MIN_AVG_DOLLAR_VOLUME_USD=20000000
MIN_PRICE_USD=10
```

`DISCORD_GUILD_ID`는 비워둘 수 있습니다. 다만 비워두면 슬래시 명령어가 디스코드에 반영되기까지 시간이 걸릴 수 있습니다. 빠르게 테스트하려면 디스코드 서버 ID를 넣는 것이 좋습니다.

`DEFAULT_TICKERS`는 최초 후보군이자 `/미너비니 후보군 작업:초기화`의 기준입니다. 명령어로 바꾼 후보군은 `data/settings.json`이 우선하며, 이 로컬 파일은 자동으로 생성됩니다.

`GROQ_API_KEY`는 `/미너비니 추세분석`에서만 사용합니다. 키가 없어도 종목추천, 후보군 관리, 진단, 검증과 사후검증은 정상 작동합니다. Groq 무료 플랜의 요청 또는 토큰 한도를 넘으면 잠시 기다리거나 한도가 초기화된 뒤 다시 실행해야 합니다.

## 디스코드 봇 연결

봇 애플리케이션 생성, 토큰 발급과 서버 설치는 Discord의 [공식 봇 시작 가이드](https://docs.discord.com/developers/quick-start/getting-started)를 참고하세요. 발급받은 토큰을 `.env`의 `DISCORD_BOT_TOKEN`에 넣으면 됩니다.

다른 서버에 설치할 때는 Discord Developer Portal의 `Installation` 설정에서 길드 설치를 활성화하고 다음 범위와 권한을 부여합니다.

- Scopes: `applications.commands`, `bot`
- Permissions: 채널 보기, 메시지 보내기, 파일 첨부

후보군 관리 명령은 실행하는 사용자에게 서버 관리 권한이 있어야 합니다. 여러 서버에서 같은 슬래시 명령을 사용하려면 `DISCORD_GUILD_ID=`를 비워 전역 명령으로 동기화합니다.

## 실행 방법

```powershell
python -m src.bot
```

또는 Windows에서 더블클릭으로 실행하려면:

```text
start_bot.bat
```

`start_bot.bat` 창이 켜져 있는 동안 봇이 작동합니다. 창을 닫으면 봇도 꺼집니다.

봇이 정상 실행되면 디스코드 서버에서 다음 명령어를 사용할 수 있습니다.

```text
/미너비니 종목추천
/미너비니 후보군 작업:조회
/미너비니 진단 티커:NVDA
```

## 디스코드 출력 형식

모든 주요 리포트는 디스코드 Markdown 위계를 공통으로 사용합니다.

- `#`: 보고서 제목
- `##`: 핵심 지표, 종목별 결과, 판정 기준 같은 주요 섹션
- `###`: 종목명, 요약, 모델명, 안내 같은 세부 제목
- `>`: 핵심 판단, 요약과 투자자문 아님 안내
- `-`: 지표, 판정 근거와 위험 요인
- 코드 블록: 복사할 티커 목록과 설정 예시

종목추천, 후보군 필터, 개별 진단, LLM 추세분석, 검증과 사후검증에 같은 형식을 적용해 명령마다 화면 구조가 달라지지 않도록 했습니다.

## 예시 출력

```markdown
# NVDA | LLM 추세 분석

### 분석 모델
`openai/gpt-oss-120b` (Groq)

## 한줄 판단
> 장기 상승 추세는 유지되지만 거래량 확인이 필요합니다.

## 추세 근거
- 현재가가 50일·150일·200일 이동평균선 위에 있습니다.
- 200일 이동평균선이 상승 중입니다.

## 반전·무효화 신호
- 현재가가 50일 이동평균선을 이탈하는지 확인합니다.

## 리스크
### 주요 위험
- 거래량 약화
- 52주 고점 부근의 매도 압력

### 안내
> 입력된 가격·거래량·재무 보조 지표의 해석이며 투자자문이나 미래 예측이 아닙니다.
```
